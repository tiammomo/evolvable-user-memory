HARD INVARIANTS

不可突破的边界

  • 01

    租户与用户隔离每个有状态操作都携带明确 Scope。

  • 02

    历史不可改写观察、修订、轨迹与结果只追加。

  • 03

    反馈必须可归因Outcome 必须引用包含该 Revision 的 Trace。

  • 04

    召回保持中性单纯读取不改变信念或效用。
